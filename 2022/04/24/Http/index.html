HTTP

  • http

http组成

  1. 请求行
  2. 首部(header请求,响应头)
  3. 实体
    GTE /images/logo.png HTTP/1.1

首部

  • 通用首部
    Cache-control 控制缓存的行为
    Date 创建报文的时间
    Pragma 报文指令
    Via 代理服务器的相关信息
    Transfer- 传输编码方式
    Connection 浏览器想要有限使用的连接类型

  • 请求的首部
    Access 能正确接收的媒体类型
    Access-Charset 能正确接收的字符集
    Access-Encoding 能正确接收的编码格式
    Host 服务端的域名
    Accept-Language 能正确接收的的语言
    User-Agent 客户端的信息

  • 响应的首部
    Accept-Ranges 是否支持某些种类
    Age 资源在代理缓存中的存在时间
    Etag 资源标识
    Location 客户端需要重定向到某个url

实体

  • 实体首部
    Allow 资源的正确请求方式
    Expires 内容的过期时间
    Content-Type 内容的媒体类型
    Last_modified 内容最后的修改时间

常见的状态码

  • 2xx
    200
    204 请求成功，但是响应的报文不包含实体的主体部分
    205 请求成功，但是响应的报文不包含实体的主体部分，但是要求请求方重置内容
    206 范围请求
  • 3xx 重定向
    301 资源永久重定向
    302 临时重定向
    303 资源存在另一个url
    304 服务器允许访问资源，发起请求条件未满足
    307 临时，保持请求方法不变向新的地址发出请求
  • 4xx 客户端错误
    400 请求报文语法错误
    401 发送的请求没有http认证信息
    403 直接被拒绝（跨域）
    404 在服务器没找到资源
  • 5xx 服务器错误(找后端)
    500 服务器发送错误
    501 后端不支持
    503 服务器崩溃超负荷运作

TLS协议

TSL协议是在传输层之上，应用层之下的协议
HTTPS还是通过HTTP协议来传输信息，但是信息通过TSL协议进行加密

TSL协议的两种加密方式：

  1. 对称加密
  2. 非对称加密

  • 对称加密：客户端和服务端两边拥有相同的密钥，两边都知道如何将密文进行加密和解密，
    这种方式很方便问题在于如何让双方都知道这个密钥，因为传输数据用的是网络，如果将密钥
    通过网络的方式进行传递的话，一旦密钥被截获那么加密就没有效果。

  • 非对称加密：有公钥和私钥之分，公钥可以让所有人知道，用于数据加密，但是要解密数据必须使用私钥，私钥只有分发公钥的一方才知道

  • 流程：假如A是服务端，B是客户端
  1. A创建公钥和私钥，将公钥公布出去让B知道
  2. B创建一个密钥，然后通过公钥加密这个密钥，并发送A
  3. A通过私钥解密这个密钥
  4. 两端就都知道密钥了

HTTP3

HTTP1中存在一些性能问题，因为浏览器限制了同一个域名下的请求数量(chrome一般是6个)，如果一个页面需要大量的请求，那么就会造成请求阻塞，需要前6个请求完才能请求下一波的6个

HTTP2解决了这些性能问题，引入了多路复用技术(一个TCP连接里面可以传输多条数据流)，采用二进制编码，大幅度提升网页性能，又因为使用了多路复用，当某个TCP连接出现丢包的情况，就会导致HTTP2的情况变复杂，反倒不如HTTP1了

HTTP3基于UDP协议的QUIC协议，解决的TCP的问题

post 和 get 的区别

副作用：服务器上的资源发送改变
幂等：服务器上的资源状态是否一致

  1. get度用于无副作用，幂等的场景，比如搜索关键字。post多用于副作用，不幂等的场景，比如注册
  2. get 请求能缓存，post不能
  3. post相对于get稍微安全一点，因为get请求的参数都包含在url里面（body也能）,并且会被浏览器保存历史记录，post不会
  4. 因为浏览器对url有长度限制，所以会影响get请求
  5. post支持更多的编码类型且不对数据类型做限制

从输入URL到页面渲染发生了什么

  1. 客户端发起DNS解析，得到服务端的IP地址
  2. TCP的三次握手，建立连接
  3. 数据传输
  4. 浏览器获取到html，css，js这种资源
  5. 浏览器会将HTML解析成DOM树，css解析成cssOM树，在合成render树
  6. 拿着render树去调用浏览器的GUI绘图功能，绘制出页面（回流重绘）
